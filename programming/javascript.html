JavaScript (ECMAScript)

JavaScript is the language of browsers: every internet browser more-or-less supports it in some way. The general trend with web developers is to hate JavaScript with no remorse, but I quite like JavaScript and find it a lovely scripting language with wonderful scripting features, just plagued with the syntax of Java: first-class functions, closures, prototypes, and curly braces and semicolons everywhere! Besides the syntax, a few other minor differences in core features (for example, the presence of IO and multithreading), and variable scoping, JavaScript is almost identical to Lua, which I am not ashamed to say is my absolute favorite language. Technically speaking, JavaScript and Java have nothing in common beside the name, which was probably used as a marketing ploy in the wake of the Java Hype of the late 1990s.
JavaScript itself isn't just a language but an implementation of the ECMAScript Specification: specifically Mozilla's implementation. There are various other implementations of ECMAScript that run on engines, internal code to interpret and execute JavaScript code. Mozilla's engine is SpiderMonkey, the first ECMAScript engine created and technically the only one that runs JavaScript. Every major browser has its own engine: Chrome uses V8, Opera uses Chakra, and Internet Explorer uses JSctipt among others.
