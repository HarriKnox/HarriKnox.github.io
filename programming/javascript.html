JavaScript

JavaScript is the language of browsers: every internet browser more-or-less supports it in some way. The general trend with web developers is to hate JavaScript, but I quite like JavaScript and find it a lovely scripting language with the Script features of Lua stuck with the Java syntax of Java: first-class functions, closures, prototypes, and curly braces and semicolons everywhere! Besides the syntax and a few other minor differences, JavaScript is very much like Lua, which I am not ashamed to say is my absolute favorite language.
Technically speaking, JavaScript and Java have nothing in common beside the name, which was probably used as a marketing ploy in the wake of the Java Hype of the late 1990's; JavaScript itself is owned by Mozilla, but the name is trademarked by Oracle (owners of Java). JavaScript is also technically not just a language, but one of the many implementations of the ECMAScript Specification: there are also JScript (Microsoft's implementation for Internet Explorer, not to be confused with JavaScript), and ActionScript. From what I can tell, JavaScript, JScript, and ActionScript are all considered the implementations and differ in copyright ownership and the list of ECMAScript standard features they support, but those differences also apply with SpiderMonkey, V8, Rhino, and Chakra. I don't understand why there is a distinction between the implementation and the engines; as far as I can tell, each engine is an implementation of an implementation.
Every internet browser has a JavaScript engine (internal code to interpret and execute JavaScript), which really is an ECMAScript engine. Mozilla owns and develops SpiderMonkey, the first JavaScript engine. There's also V8 (for Chrome, Safari, and other WebKit browsers) and Chakra (for Opera) and quite a few other engines for other browsers, command-line interpreters, and embedded scripting functionality in programs.
